Introducción a Arduino

Arduino es una plataforma de creación de electrónica de código abierto, la cual está basada en hardware y software libre, flexible y fácil de utilizar para los creadores y desarrolladores.

¿Qué es Arduino? Arduino es una plataforma de desarrollo basada en una placa electrónica con un microcontrolador y un entorno de desarrollo (IDE), diseñada para facilitar el uso de la electrónica en proyectos multidisciplinares.

Características Principales

  • Hardware libre: Los diseños de las placas están disponibles bajo licencia de código abierto
  • Software libre: El IDE y las bibliotecas son de código abierto
  • Multiplataforma: El software funciona en Windows, macOS y Linux
  • Entorno sencillo: Fácil de usar para principiantes, pero flexible para usuarios avanzados
  • Bajo costo: Las placas Arduino son relativamente económicas
  • Comunidad activa: Gran comunidad de usuarios que comparten conocimientos y proyectos
  • Extensible: Posibilidad de añadir shields (placas de expansión)
  • Variedad de placas: Diferentes modelos para distintas necesidades
  • Programación sencilla: Basada en C/C++, pero simplificada
  • Documentación amplia: Tutoriales, ejemplos y referencias disponibles

Historia y Evolución

Arduino nació en 2005 en el Instituto de Diseño Interactivo de Ivrea (Italia) como un proyecto para estudiantes. Sus creadores, Massimo Banzi, David Cuartielles, Tom Igoe, Gianluca Martino y David Mellis, buscaban una herramienta sencilla y económica para que los estudiantes pudieran crear proyectos de electrónica sin necesidad de conocimientos avanzados.

Desde entonces, Arduino ha evolucionado considerablemente:

  • 2005: Primera placa Arduino (Serial)
  • 2008: Arduino Duemilanove, una de las primeras placas ampliamente adoptadas
  • 2010: Arduino Uno, que se convirtió en la placa estándar
  • 2012: Arduino Leonardo, con capacidades USB mejoradas
  • 2014: Arduino Yún, con conectividad WiFi
  • 2015: Arduino MKR1000, enfocada en IoT
  • 2018: Arduino Nano 33 IoT, con Bluetooth y WiFi en formato compacto
  • 2020: Arduino Portenta H7, para aplicaciones industriales

Ventajas

Hardware de Arduino

El ecosistema de hardware Arduino incluye diversas placas con diferentes características, capacidades y formatos, adaptadas a distintas necesidades y aplicaciones.

Placas Arduino Más Comunes

Arduino UNO

El Arduino UNO es la placa más popular y utilizada, ideal para principiantes y proyectos generales. Se basa en el microcontrolador ATmega328P.

Características principales:
  • Microcontrolador: ATmega328P
  • Voltaje de operación: 5V
  • Voltaje de entrada (recomendado): 7-12V
  • Pines digitales I/O: 14 (6 con PWM)
  • Pines analógicos de entrada: 6
  • Memoria Flash: 32 KB
  • SRAM: 2 KB
  • EEPROM: 1 KB
  • Frecuencia de reloj: 16 MHz
Usos comunes:
  • Proyectos de iniciación
  • Control de luces y displays
  • Lectura de sensores
  • Automatización básica
Arduino NANO

El Arduino NANO es una versión compacta del Arduino UNO, ideal para proyectos donde el espacio es limitado. También se basa en el microcontrolador ATmega328P.

Características principales:
  • Microcontrolador: ATmega328P
  • Voltaje de operación: 5V
  • Voltaje de entrada (límite): 6-20V
  • Pines digitales I/O: 14 (6 con PWM)
  • Pines analógicos de entrada: 8
  • Memoria Flash: 32 KB
  • SRAM: 2 KB
  • EEPROM: 1 KB
  • Frecuencia de reloj: 16 MHz
Usos comunes:
  • Wearables
  • Proyectos embebidos
  • Robótica móvil
  • Sensores remotos
Arduino MEGA

El Arduino MEGA es una placa con mayor capacidad de procesamiento y más pines I/O que el Arduino UNO, ideal para proyectos complejos que requieren muchas conexiones. Se basa en el microcontrolador ATmega2560.

Características principales:
  • Microcontrolador: ATmega2560
  • Voltaje de operación: 5V
  • Voltaje de entrada (recomendado): 7-12V
  • Pines digitales I/O: 54 (15 con PWM)
  • Pines analógicos de entrada: 16
  • Memoria Flash: 256 KB
  • SRAM: 8 KB
  • EEPROM: 4 KB
  • Frecuencia de reloj: 16 MHz
Usos comunes:
  • Impresoras 3D
  • Robótica avanzada
  • Sistemas de control complejos
  • Proyectos con muchos sensores y actuadores

Shields (Placas de Expansión)

Los shields son placas que se conectan directamente a las placas Arduino para añadir funcionalidades específicas, como:

  • Shield Ethernet: Conexión a redes Ethernet
  • Shield WiFi: Conexión a redes WiFi
  • Shield Motor Driver: Control de motores DC y paso a paso
  • Shield LCD: Interfaz con pantallas LCD
  • Shield Sensor: Integración de múltiples sensores
  • Shield Relé: Control de dispositivos de alta potencia

Programación de Arduino

La programación de Arduino se realiza mediante el entorno de desarrollo (IDE) de Arduino, que proporciona un editor de código, un compilador y herramientas para cargar el programa en la placa.

Entorno de Desarrollo (IDE)

El IDE de Arduino es una aplicación multiplataforma escrita en Java que incluye:

  • Editor de código con resaltado de sintaxis
  • Compilador integrado
  • Herramienta de carga de programas a la placa
  • Monitor serial para comunicación con la placa
  • Gestor de bibliotecas y placas
  • Ejemplos preinstalados

Existen dos versiones principales del IDE:

  • IDE 1.x: La versión clásica y más utilizada
  • IDE 2.x: La nueva versión con interfaz modernizada y más funcionalidades

Estructura Básica de un Programa

Estructura básica de un sketch de Arduino
// Declaraciones globales
int ledPin = 13;  // Variable global

// Función de configuración - se ejecuta una vez al inicio
void setup() {
  // Inicialización de pines, comunicación serial, etc.
  pinMode(ledPin, OUTPUT);  // Configura el pin como salida
  Serial.begin(9600);       // Inicia comunicación serial a 9600 baudios
}

// Función principal - se ejecuta continuamente en bucle
void loop() {
  // Código que se ejecuta repetidamente
  digitalWrite(ledPin, HIGH);  // Enciende el LED
  delay(1000);                 // Espera 1 segundo
  digitalWrite(ledPin, LOW);   // Apaga el LED
  delay(1000);                 // Espera 1 segundo
}

// Funciones adicionales definidas por el usuario
void miFuncion() {
  // Código de la función
}

Todo programa de Arduino (llamado "sketch") debe contener al menos dos funciones:

  • setup(): Se ejecuta una sola vez al inicio y se utiliza para configurar pines, inicializar comunicaciones, etc.
  • loop(): Se ejecuta continuamente después de setup() y contiene el código principal del programa.

Funciones Básicas

Entrada/Salida Digital
  • pinMode(pin, mode): Configura un pin como entrada o salida
  • digitalWrite(pin, value): Escribe un valor HIGH o LOW en un pin digital
  • digitalRead(pin): Lee el valor de un pin digital (HIGH o LOW)
// Ejemplo de uso
pinMode(13, OUTPUT);       // Configura el pin 13 como salida
digitalWrite(13, HIGH);    // Establece el pin 13 en estado alto (5V)
int valor = digitalRead(7); // Lee el estado del pin 7
Entrada/Salida Analógica
  • analogRead(pin): Lee el valor de un pin analógico (0-1023)
  • analogWrite(pin, value): Escribe un valor PWM en un pin (0-255)
// Ejemplo de uso
int sensorValue = analogRead(A0);  // Lee el valor del sensor en A0 (0-1023)
analogWrite(9, 128);               // Establece el pin 9 al 50% del ciclo de trabajo

Nota: La función analogWrite() no genera una verdadera salida analógica, sino una señal PWM (Modulación por Ancho de Pulso).

Tiempo
  • delay(ms): Pausa el programa durante los milisegundos especificados
  • millis(): Devuelve el número de milisegundos desde que el programa comenzó a ejecutarse
  • micros(): Devuelve el número de microsegundos desde que el programa comenzó a ejecutarse
  • delayMicroseconds(us): Pausa el programa durante los microsegundos especificados
// Ejemplo de uso
delay(1000);                  // Pausa el programa durante 1 segundo
unsigned long tiempo = millis(); // Obtiene el tiempo actual en milisegundos
Comunicación Serial
  • Serial.begin(baudRate): Inicia la comunicación serial a la velocidad especificada
  • Serial.print(data): Envía datos al puerto serial
  • Serial.println(data): Envía datos al puerto serial y añade un salto de línea
  • Serial.available(): Devuelve el número de bytes disponibles para leer
  • Serial.read(): Lee un byte de datos entrantes
// Ejemplo de uso
Serial.begin(9600);           // Inicia comunicación a 9600 baudios
Serial.println("Hola Mundo"); // Envía texto con salto de línea
if (Serial.available() > 0) {
  int incomingByte = Serial.read(); // Lee un byte entrante
}

Bibliotecas

Las bibliotecas extienden la funcionalidad de Arduino, proporcionando código reutilizable para interactuar con hardware específico o implementar funcionalidades complejas.

Bibliotecas Estándar
  • EEPROM: Para leer y escribir en la memoria EEPROM
  • SPI: Para comunicación con dispositivos SPI
  • Wire: Para comunicación I2C
  • Servo: Para control de servomotores
  • LiquidCrystal: Para pantallas LCD
  • SD: Para tarjetas SD
Uso de Bibliotecas
// Incluir biblioteca
#include <Servo.h>

// Crear objeto
Servo miServo;

void setup() {
  // Asociar servo al pin 9
  miServo.attach(9);
}

void loop() {
  // Mover servo a 90 grados
  miServo.write(90);
  delay(1000);
}

Ejemplos Prácticos

A continuación se presentan algunos ejemplos básicos para comenzar a programar con Arduino.

Parpadeo de LED (Blink)

Este es el ejemplo más básico y sirve como "Hola Mundo" en Arduino. Hace parpadear un LED conectado al pin 13 (que en muchas placas Arduino tiene un LED integrado).

Blink.ino
/*
  Blink
  Enciende un LED durante un segundo, luego lo apaga durante un segundo, repetidamente.
*/

// El LED está conectado al pin 13 en la mayoría de placas Arduino
const int ledPin = 13;

// La función setup se ejecuta una vez cuando presionas reset o enciendes la placa
void setup() {
  // Inicializa el pin digital como una salida
  pinMode(ledPin, OUTPUT);
}

// La función loop se ejecuta repetidamente para siempre
void loop() {
  digitalWrite(ledPin, HIGH);   // Enciende el LED (HIGH es el nivel de voltaje)
  delay(1000);                  // Espera un segundo
  digitalWrite(ledPin, LOW);    // Apaga el LED haciendo el voltaje LOW
  delay(1000);                  // Espera un segundo
}
Diagrama de Conexión
Explicación
  • Definimos el pin 13 como salida en la función setup()
  • En el loop(), alternamos entre encender (HIGH) y apagar (LOW) el LED
  • Usamos delay() para crear pausas de 1 segundo entre cada cambio
Lectura de Sensor Analógico

Este ejemplo lee el valor de un sensor analógico (como un potenciómetro o un sensor de luz) conectado al pin A0 y muestra el valor en el Monitor Serial.

AnalogRead.ino
/*
  Lectura Analógica
  Lee el valor de un pin analógico y lo muestra en el Monitor Serial.
*/

const int sensorPin = A0;  // Pin analógico para el sensor
int sensorValue = 0;       // Variable para almacenar el valor del sensor

void setup() {
  // Inicializa la comunicación serial a 9600 bits por segundo
  Serial.begin(9600);
}

void loop() {
  // Lee el valor del sensor
  sensorValue = analogRead(sensorPin);
  
  // Imprime el valor en el Monitor Serial
  Serial.print("Valor del sensor: ");
  Serial.println(sensorValue);
  
  // Pequeña pausa para no saturar el Monitor Serial
  delay(100);
}
Diagrama de Conexión
Explicación
  • Inicializamos la comunicación serial a 9600 baudios
  • Leemos el valor del sensor con analogRead(), que devuelve un valor entre 0 y 1023
  • Enviamos el valor al Monitor Serial para visualizarlo
  • Esperamos 100ms antes de la siguiente lectura
Control de Servo

Este ejemplo muestra cómo controlar un servomotor utilizando la biblioteca Servo. El servo se moverá de 0 a 180 grados y viceversa.

ServoSweep.ino
/*
  Servo Sweep
  Mueve un servomotor de 0 a 180 grados y viceversa.
*/

#include <Servo.h>  // Incluye la biblioteca Servo

Servo miServo;  // Crea un objeto servo para controlar el servomotor
int pos = 0;    // Variable para almacenar la posición del servo

void setup() {
  miServo.attach(9);  // Asocia el servo al pin 9
}

void loop() {
  // Mueve el servo de 0 a 180 grados
  for (pos = 0; pos <= 180; pos += 1) {
    miServo.write(pos);  // Establece la posición del servo
    delay(15);           // Espera 15ms para que el servo llegue a la posición
  }
  
  // Mueve el servo de 180 a 0 grados
  for (pos = 180; pos >= 0; pos -= 1) {
    miServo.write(pos);  // Establece la posición del servo
    delay(15);           // Espera 15ms para que el servo llegue a la posición
  }
}
Diagrama de Conexión
Explicación
  • Incluimos la biblioteca Servo y creamos un objeto Servo
  • Asociamos el servo al pin 9 en setup()
  • Usamos dos bucles for para mover el servo de 0 a 180 grados y viceversa
  • La función write() establece la posición del servo en grados
Sensor de Temperatura con LCD

Este ejemplo más avanzado lee la temperatura de un sensor LM35 y la muestra en una pantalla LCD.

TemperaturaLCD.ino
/*
  Sensor de Temperatura con LCD
  Lee la temperatura de un sensor LM35 y la muestra en una pantalla LCD.
*/

#include <LiquidCrystal.h>  // Incluye la biblioteca LCD

// Inicializa la biblioteca LCD con los números de los pines de interfaz
LiquidCrystal lcd(12, 11, 5, 4, 3, 2);

// Pin analógico para el sensor de temperatura LM35
const int sensorPin = A0;

void setup() {
  // Configura el LCD con 16 columnas y 2 filas
  lcd.begin(16, 2);
  
  // Muestra un mensaje inicial
  lcd.print("Temperatura:");
}

void loop() {
  // Lee el valor del sensor
  int sensorValue = analogRead(sensorPin);
  
  // Convierte la lectura analógica (0-1023) a voltaje (0-5V)
  float voltage = sensorValue * (5.0 / 1023.0);
  
  // Convierte el voltaje a temperatura en grados Celsius (LM35: 10mV por grado)
  float temperature = voltage * 100.0;
  
  // Posiciona el cursor en la primera columna, segunda fila
  lcd.setCursor(0, 1);
  
  // Muestra la temperatura
  lcd.print(temperature);
  lcd.print(" C        ");  // Espacios para borrar caracteres anteriores
  
  // Espera un segundo
  delay(1000);
}
Diagrama de Conexión
Explicación
  • Incluimos la biblioteca LiquidCrystal y configuramos los pines
  • Inicializamos el LCD con 16 columnas y 2 filas
  • Leemos el valor del sensor y lo convertimos a voltaje
  • Convertimos el voltaje a temperatura (el LM35 proporciona 10mV por grado Celsius)
  • Mostramos la temperatura en la segunda fila del LCD

Consejo: El IDE de Arduino incluye muchos más ejemplos en el menú Archivo > Ejemplos. Estos ejemplos cubren desde conceptos básicos hasta funcionalidades avanzadas y son una excelente manera de aprender.
